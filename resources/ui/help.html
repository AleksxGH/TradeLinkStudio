About TradeLink Studio

TradeLink Studio is a desktop application for computing and analyzing network influence indices.

The application supports project creation and data input via .xlsx and .csv files, including adjacency matrices and quota lists.
Input data can be edited directly within the application.

The system performs computation of Copeland, Bundle, and Pivotal indices in both weighted and unweighted variants, along with their normalized shares.

All calculation results are displayed instantly inside the application, allowing for analysis of computed metrics.
Results can be exported as Excel tables for further use.

Index Calculations

Copeland Index is the sum of incoming edge weights for each vertex.

Bundle Index counts critical subsets of incoming neighbors of size from 1 to k. A subset is critical if its total weight is greater than or equal to the vertex quota.

Pivotal Index counts pivotal vertices inside each critical subset. A vertex is pivotal if removing it makes the subset non-critical. Each pivotal occurrence adds 1.

Pivotal Weighted Index uses the same pivotal subsets, but each pivotal occurrence adds the size of the subset instead of 1.

Normalization

normalized_value = value / sum(all values of same index).

If the total sum is zero, all normalized values are shown as zero.

Input Data Format: XLSX

The loader reads fixed positions:

Cell LocationContentType
B1 (row 0, col 2)Number of vertices (optional)Integer (N)
D2 (row 1, col 3)Subset sizeInteger (k)
F2 onwards (row 1, col 6...)QuotasN float values
B3 onwards (row 2, col 1...)Horizontal vertex namesN string values
A4 onwards (row 3+, col 0)Vertical vertex namesN string values
B4 to (N+3, N+1) (rows 3+, cols 1+)Adjacency matrixNxN matrix

Decimal numbers are written separated by commas (,)

The numbers on the diagonal are either absent or all equal to zero.

Input Data Format: CSV

The CSV example is shown exactly by rows. Vertex names are in the first row (horizontal) with quota as the last column.

CSV Content
v1;v2;v3;quota
0;2,5;1,2;4,0
1,0;0;3,1;3,5
4,2;2,0;0;2,8

Format requirements:

  • Header row: vertex names, ending with quota label
  • Data rows: N weights and quota value
  • Matrix must be square: N vertices and N data rows
  • Empty cells in matrix are treated as 0
  • Delimiter can be semicolon or comma (detected automatically)
  • Decimal comma and decimal point are both accepted
  • Subset size in CSV is auto-calculated as max(1, N-1)
