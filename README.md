# TradeLink Studio

## 1. О проекте

TradeLink Studio — настольное приложение на Python + PyQt5 для анализа ориентированных взвешенных графов и расчета индексов влияния/критичности вершин.

Приложение ориентировано на сценарий, где:
- есть набор вершин;
- есть матрица весов влияния между вершинами (дуг графа);
- для каждой вершины задана квота (порог);
- задается максимальный размер подмножества k (subset size);
- рассчитываются и сравниваются несколько индексов.

Текущая версия: v.0.3-alpha.

## 2. Ключевые возможности

- Создание и открытие проектов.
- Хранение проектов по умолчанию в системной папке ProgramData.
- Поддержка пользовательских (внешних) путей проектов.
- Редактируемая матрица весов (QTableView).
- Редактируемые квоты по вершинам.
- Управление размерностью матрицы:
  - Expand Matrix;
  - Shrink Matrix;
  - Rename Vertices.
- Настройка параметров расчета:
  - subset size (k);
  - precision (число знаков после запятой);
  - отображение PI' weighted;
  - отображение нормализованных колонок.
- Расчет индексов:
  - Copeland;
  - Bundle;
  - Pivotal;
  - PI' (взвешенная версия pivotal).
- Нормализация индексов (доли, сумма = 1).
- Undo/Redo по снапшотам состояния.
- Загрузка исходных данных из XLSX или CSV.
- Экспорт результата в XLSX в формате, совместимом с загрузчиком.
- Локальное логирование ошибок и технических событий.

## 3. Архитектура

Проект разделен на несколько уровней:

- UI слой:
  - HomeWindow: домашний экран, список проектов, настройки, помощь.
  - MainWindow: рабочее окно проекта, таблицы, вычисления, undo/redo.
- Service слой:
  - ConfigManager: загрузка/сохранение конфигурации пользователя.
  - ProjectManager: сохранение/загрузка проекта (project.json), сериализация графа.
  - DataController: операции над состоянием и запуск пересчета.
  - AppPaths: вычисление системных путей хранения.
  - LoggingService: логирование и очистка логов.
  - Validators: валидация имен проекта и вершин.
- Core слой:
  - DataStore: история состояний, current index, дедупликация снапшотов.
  - ComputeEngine: вычислительный движок.
  - IndicesCalculator: реализация индексов и нормализации.
  - Project: модель проекта.
- Data слой:
  - loader.py: чтение входных форматов XLSX и CSV.
  - exporter.py: сохранение итогов в XLSX.

## 4. Вычислительная модель

### 4.1. Граф

Используется networkx.DiGraph.

- Вершины: элементы множества V.
- Дуги: ориентированные связи с весом weight.
- Матрица смежности: числовая, допускает 0 (отсутствие дуги).

### 4.2. Параметры

- quotas: словарь порогов для вершин.
- subset_size (k): максимальный размер перебираемых групп входящих ребер.

### 4.3. Индексы

- Copeland index:
  - сумма весов входящих дуг вершины.

- Bundle index:
  - количество критических групп входящих ребер размером от 1 до k,
  - где сумма весов группы >= quota целевой вершины.

- Pivotal index:
  - число pivotal-участий в критических группах,
  - ребро считается pivotal, если группа критична, а без этого ребра перестает быть критичной.

- PI' (weighted pivotal):
  - как Pivotal, но вклад учитывает размер группы.

- Нормализованные версии:
  - значение / сумма по индексу;
  - если сумма 0, возвращаются нули.

## 5. Undo/Redo и история состояния

Состояния хранятся в DataStore:

- states: список снапшотов.
- current_index: текущая позиция в истории.
- push(state):
  - копирует состояние (deep copy),
  - обрезает "ветку вперед" при новой записи,
  - пропускает дубликаты подряд через fingerprint.
- undo()/redo():
  - сдвигают current_index и возвращают текущее состояние.

Fingerprint учитывает:
- порядок вершин;
- subset_size;
- quotas;
- decimal_precision;
- флаги видимости индексов;
- содержимое матрицы в детерминированном порядке.

## 7. Формат входных данных (XLSX и CSV)

Приложение поддерживает два формата импорта: XLSX и CSV.

### 7.1. Формат XLSX

Загрузчик ожидает фиксированный layout:

- [0,1]: число вершин N
- [1,3]: subset size
- [1,5..]: квоты (N значений)
- [2,1..]: имена вершин
- [3..3+N, 1..1+N]: матрица весов

Примечания:
- NaN в матрице заменяются на 0.

### 7.2. Формат CSV

CSV должен быть в формате с заголовком и матрицей:

```
v1, v2, v3, quota
v1, w11, w12, w13, q1
v2, w21, w22, w23, q2
v3, w31, w32, w33, q3
```

Описание:
- Первая строка содержит имена вершин (до колонки "quota").
- Последующие строки содержат:
  - Первая ячейка: имя вершины (исходящий узел).
  - Следующие N ячеек: веса входящих дуг для матрицы.
  - Последняя ячейка: квота для вершины.
- **Матрица должна быть квадратной**: количество строк данных = количество вершин в заголовке.
- **Каждая строка должна содержать**: имя вершины + N весов + квота (всего N+2 элемента).

Пример с реальными данными:

```
Supplier_A, Supplier_B, Supplier_C, Supplier_D, quota
Supplier_A, 0, 2.5, 1.2, 3.0, 4.0
Supplier_B, 2.0, 0, 1.5, 2.5, 3.5
Supplier_C, 1.5, 1.0, 0, 2.0, 3.0
Supplier_D, 3.5, 2.0, 2.5, 0, 5.0
```

Парсер автоматически определит subset_size как N-1 (максимальный размер подмножества = количество вершин - 1) и проверит корректность структуры матрицы.

## 8. Экспорт результата (XLSX)

Экспортер формирует таблицу, совместимую с логикой чтения:

- записывает базовые метаданные (N, subset size, quotas);
- записывает вершины и матрицу;
- добавляет колонки индексов справа от матрицы;
- в случае проблем сохранения пытается альтернативное имя с timestamp.

## 10. Валидация

### 10.1. Имя проекта

Проверяется:
- не пустое;
- не содержит запрещенных символов Windows: <>:"/\\|?*;
- не заканчивается пробелом/точкой;
- не совпадает с зарезервированными именами (CON, PRN, AUX, NUL, COMx, LPTx).

### 10.2. Имена вершин

Проверяется:
- непустые;
- без переводов строк;
- уникальные.

## 11. Логирование и обработка ошибок

## 12. Установка и запуск

## 12.1. Требования

- Python 3.12 (рекомендуется, в проекте используется 3.12.6)
- Windows 10/11

Зависимости из requirements.txt:
- PyQt5==5.15.9
- pandas==2.1.1
- numpy==1.26.1
- openpyxl==3.1.3

## 12.2. Установка

1. Создать и активировать виртуальное окружение.
2. Установить зависимости:

   pip install -r requirements.txt

3. Запустить приложение:

   python main.py

## 13. Сборка EXE (PyInstaller)

Используйте скрипт:

- build_exe.bat

Что делает скрипт:
- активирует venv (если найдено);
- очищает build/dist;
- запускает generate_build_metadata.py;
- генерирует имя exe из версии;
- вызывает pyinstaller --onefile --windowed c нужными add-data.

Дополнительно:
- generate_build_metadata.py формирует build/version_file.txt в совместимом формате для текущего PyInstaller.

## 14. Ограничения и известные особенности

- Комбинаторные индексы (Bundle/Pivotal/PI') чувствительны к росту N и k.
- Для крупных графов время расчета может быть заметным.
- Проект ориентирован на desktop workflow и локальное хранение данных.
